# Member Portal - Full SaaS Features

## 🎯 Two User Modes

### Mode 1: Discovery (Non-Members)
- Browse businesses
- Search activities
- Book drop-in classes
- Pay per session

### Mode 2: Member Portal (Active Members)
- Access their club's portal
- View their schedule
- Register for upcoming classes
- Track attendance & progress
- Communicate with coaches
- View announcements
- Manage membership

## 🗄️ Updated Database Schema

### New Tables Needed

```sql
-- Announcements
CREATE TABLE announcements (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  business_id UUID NOT NULL REFERENCES businesses(id),
  author_id UUID NOT NULL REFERENCES users(id), -- staff who posted
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  target_audience TEXT NOT NULL CHECK (target_audience IN ('all_members', 'specific_service', 'specific_members')),
  service_id UUID REFERENCES services(id), -- if targeting specific class
  is_pinned BOOLEAN DEFAULT false,
  send_push BOOLEAN DEFAULT false,
  send_email BOOLEAN DEFAULT false,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Messages (direct communication)
CREATE TABLE messages (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  business_id UUID NOT NULL REFERENCES businesses(id),
  sender_id UUID NOT NULL REFERENCES users(id),
  recipient_id UUID NOT NULL REFERENCES users(id),
  message TEXT NOT NULL,
  is_read BOOLEAN DEFAULT false,
  read_at TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Coach notes (private notes on members)
CREATE TABLE member_notes (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  business_id UUID NOT NULL REFERENCES businesses(id),
  member_id UUID NOT NULL REFERENCES users(id),
  coach_id UUID NOT NULL REFERENCES users(id),
  booking_id UUID REFERENCES bookings(id), -- which class was this about
  note TEXT NOT NULL,
  is_visible_to_parent BOOLEAN DEFAULT false, -- can parent see this?
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Achievements/Badges (gamification)
CREATE TABLE achievements (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  business_id UUID NOT NULL REFERENCES businesses(id),
  name TEXT NOT NULL,
  description TEXT,
  icon_url TEXT,
  criteria JSONB, -- {"type": "attendance", "threshold": 10}
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE member_achievements (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  achievement_id UUID NOT NULL REFERENCES achievements(id),
  member_id UUID NOT NULL REFERENCES users(id),
  child_id UUID REFERENCES children(id),
  earned_at TIMESTAMPTZ DEFAULT NOW()
);

-- Check-ins (attendance tracking)
CREATE TABLE check_ins (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  booking_id UUID NOT NULL REFERENCES bookings(id),
  checked_in_by UUID REFERENCES users(id), -- staff who checked them in
  check_in_time TIMESTAMPTZ DEFAULT NOW(),
  check_out_time TIMESTAMPTZ,
  notes TEXT
);

-- Skills/Progress tracking (optional, for sports academies)
CREATE TABLE skills (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  business_id UUID NOT NULL REFERENCES businesses(id),
  service_id UUID REFERENCES services(id),
  name TEXT NOT NULL,
  description TEXT,
  difficulty_level TEXT CHECK (difficulty_level IN ('beginner', 'intermediate', 'advanced')),
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE member_skills (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  member_id UUID NOT NULL REFERENCES users(id),
  child_id UUID REFERENCES children(id),
  skill_id UUID NOT NULL REFERENCES skills(id),
  proficiency INTEGER CHECK (proficiency >= 0 AND proficiency <= 100),
  last_assessed_at TIMESTAMPTZ,
  assessed_by UUID REFERENCES users(id),
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 🔄 Key User Flows

### Flow 1: New User → Member
```
1. Download app
2. Browse businesses
3. Find "RAK Parkour Academy"
4. View services
5. Choose plan: "2 sessions per week - $300/month"
6. Sign up + add child info
7. Complete payment
8. Now in member portal → Register for first class
```

### Flow 2: Active Member → Register for Class
```
1. Open app → See "My Clubs"
2. Tap "RAK Parkour Academy"
3. Tap "Register for Classes"
4. See this week's schedule + remaining sessions
5. Tap "Monday 4:15 PM" → [Register]
6. Confirmation + added to "My Schedule"
7. Get reminder 24h before + 1h before
```

### Flow 3: Parent Checks Progress
```
1. My Clubs → RAK Parkour Academy
2. Tap "Progress" tab
3. See:
   - Attendance this month
   - Coach notes
   - Skills progress
   - Achievements earned
```

### Flow 4: Business Sends Announcement
```
1. Business Dashboard → Communication
2. Click "New Announcement"
3. Select audience: "Kids Beginners Class"
4. Write message: "Class cancelled tomorrow"
5. Enable push + email
6. Send
7. All members get notified instantly
```

## 💡 Key Insights

**This is now a "club management platform" not just booking:**

1. **Members have a home** - They live in the app, not just book and leave
2. **Regular engagement** - Schedule, messages, progress keep them coming back
3. **Communication is key** - Coaches can engage, parents stay informed
4. **Progress matters** - Especially for kids sports, tracking growth is valuable
5. **Retention tool** - Members who engage with portal have higher retention

**Similar platforms:**
- TeamSnap (team sports management)
- TeamUp (gym/studio member portal)
- Pike13 (client engagement)
- Glofox (boutique studio software)

But simpler and focused on local UAE market.
